##java串口通信

实习工作的第一个项目就是一个针对stm32（大概是）设备的初始化工作，为他们赋予唯一的设备号id并将设备的序号保存到服务器中，并且该程序应能在pc/Android/ios中都能被使用。我的第一反应就是写一个web端的程序那么不就可以轻松的在三端上使用了吗？但是我后想了想web端中能不能进行串口通讯呢？答案是不行的，大多数的浏览器都不允许web程序对本机的资源进行直接操纵。因此我希望使用java至少先使程序能在Android与pc中使用起来，（后来我发现了electron）。

先简单的说一下java中能实现串口通信的第三方库吧，

1：rxtx.jar,我使用的就是这个库。首先先要注意的是电脑中的java版本，目前我使用过的多数java串口通讯的第三方库都是远古时代的产物，所以Java的版本不能过高，据我测试过后，我认为Java1.7/1.8版本比较靠谱，我一开始使用的是Java11（图个新鲜），结果凉了。

更改了Java版本后正常运行

官网地址：http://fizzed.com/oss/rxtx-for-java

可能需要科学上网。

2：JSerialComm，在maven上找到的最新版本为2.5.1为2019年5月更新的，是我在这几个Java串口通讯库中唯一的可以使用Java11版本环境下使用的第三方库，自认为这个库比较好用，推荐使用这个库。

地址：https://mvnrepository.com/artifact/com.fazecast/jSerialComm

3：JSSC，java-simple-serial-connector最初是在git hub上搜索java serialport项目时候看到的，上次更新事件为2014年，出现的问题与rxtx相同，不能在Java1.8以上的版本使用。由于相对比rxtx更新，JSSC支持的环境更多可用于Windows（x86，x86-64），Linux（x86，x86-64，ARM软硬浮动），Solaris（x86，x86-64），Mac OS X（x86，x86-64，PPC）的本机库，PPC64）。所有本机库都包含在jssc.jar文件中。

Github项目地址：https://github.com/scream3r/java-simple-serial-connector

我的github串口通讯demo的地址：https://github.com/LG32/java-serialComm。
