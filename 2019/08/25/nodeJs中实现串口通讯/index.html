nodeJs中实现串口通讯

nodeJs中实现串口通讯

想在一个程序中实现在多平台上串口通讯，并希望有可能的情况下在移动端中也实现串口怎么办？使用electron+nodeJs便能实现。

在最初听到需要在多平台上都可以运行时，我第一想法就是使用web进行开发，这样就能在几乎所有拥有浏览器的系统中使用了。然而使用web开发有一个最大的问题——浏览器为了保护用户本地信息的安全性都禁止了程序直接的访问本机的端口信息，唯一例外的时IE浏览器中还尚在支持的ActiveX，它可以提供更多的交互而且还能使用java进行开发。但是只支持IE浏览器明显无法达到实现在多平台下运行的这一要求。于是我选择了使用Electron-vue框架。

Electron是由Github开发，用HTML，CSS和JavaScript来构建跨平台桌面应用程序的一个开源库。 Electron通过将Chromium和Node.js合并到同一个运行时环境中，并将其打包为Mac，Windows和Linux系统下的应用来实现这一目的。通过这个框架我能使用nodeJs进行串口通信并做到多平台开发。 开发时我还添加了vue.js框架，electron-vue框架把electron和vue 做了个集合，不用用户手动去一个个搭建环境。

环境搭建

1、需要nodeJs与npm这就不说了

2、安装python2.7

安装python2.7版本，不知道谁说的electron不支持python3，网上的大多数教程都建议使用python2.7，也希望有勇士可以试试使用python3看看可不可以。(node-gyp需要)

3、安装vue-cli脚手架

在控制台输入命令

npm i -g vue-cli;

安装好脚手架后继续输入命令：

vue init simulatedgreg/electron-vue my-serial;

备注： 有可能出现'vue' 不是内部或者外部命令，也不是可运行的程序或者批处理文件。原因是：vue没有加入到环境变量中。不过后面自己加的时候也是有问题，后面，我自己很粗暴的复制出了vue的安装路径（上一个命令安装完成之后直接有对应的路径，复制即可），来进行安装。

如：C:\Users\xxxxxx\npm\vue init simulatedgreg/electron-vue my-serial;

完成了安装之后，输入项目信息（vue项目在初始化的时候要填一堆信息，不想研究就一路回车就好了）。

完事后会提示如下图的信息。

图是网上扒的文件名按自己建的文件名改一下，按照提示继续进行输入

随即便能启动模板程序

4、安装serialport

命令 npm i –save serialport

没vpn网速慢可使用cnpm i –save serialport

安装electron-rebuild, 因为serialport是根据系统环境编译的，当我们安装时，我们编译的成的是系统环境的serialport，所以我们需要重新编译成electron的环境，所以我们需要electron-rebuild

cnpm install –save-dev electron-rebuild

重新编译, 因为我当前版本的electron是2.0.4的，所以我们重新把模块编译成适应2.0.4的, 这里记得要根据你的electron版本编译

./node_modules/.bin/electron-rebuild -v 2.0.4
